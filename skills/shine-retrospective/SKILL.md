---
name: shine-retrospective
description: "Run a structured retro on a delivered project — what worked / what didn't / what to systemize into a memory entry."
argument-hint: "<project or client name>"
allowed-tools:
  - Read
  - Write
  - Glob
---

<objective>
Guide a retro conversation and persist the learnings:

1. Retrieve project timeline from memory + session logs.
2. Ask the user: wins, friction, surprises, one thing to change.
3. Classify each into: `preference` (personal working style) / `client` (specific to this account) / `process` (repeatable).
4. Write a new memory entry for anything reusable.
5. Output a 1-page "lessons learned" markdown for the user to share internally.
</objective>
